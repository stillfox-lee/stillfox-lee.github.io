k8s 提供的分布式选主

之前在学习 MIT6.824 的时候，就会涉及到 Raft 协议，当时在实现选举逻辑的时候也折腾过几个晚上。在学习 k8s 的过程中，发现基于 API Server 就可以完成一个多节点选主的需求。所以就花了点时间看看 k8s 是怎么实现的。

其实这也是 k8s 的一个厉害之处，基于 k8s可以快速地实现复杂的分布式算法逻辑。在性能要求不是特别严苛（数据库之类应用）的场景下，用极低的开发成本可以为你带来一个工作良好的特性；等性能真正成为瓶颈的时候，再优化也无妨。毕竟当年 Golang 发布的时候，它的调度器也是一个刚刚能用的状态。
